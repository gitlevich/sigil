# Ontology Browser

The left panel. Shows the emerging @sigil tree as I narrate. Each node is a @sigil; I can #navigate to it, #rename it, #move it, or #delete it. I can see at a glance which sigils have affordances and invariants defined, and which are still bare names.

I can #search to filter the tree by name. I can #add-peer to create a new sigil next to the selected one. I can #drag to rearrange the tree.

Each node has a #definition — a one-line summary that I can edit inline. The definition is what the @DesignPartner sees when reasoning about the sigil without reading its full language.

What I see must match what exists: !structural-truth is non-negotiable.

Each node is decorated with #affordance-icon for each @affordance and #invariant-icon for each @invariant it declares. On hover, they show their names.
