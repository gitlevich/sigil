@SigilFolder is part of @SigilSpecContent under Application, not Memory: per @SigilSpecContent/MemoryModelSeparation, it is only the filesystem and on-disk representation of a sigil, used for persistence, editing, storage, and transfer, not the source of a sigil’s experiential memory or conceptual identity. A @SigilFolder contains the sigil’s @LanguageFile, @AffordanceFiles, @InvariantFiles, and child @SigilFolders, storing child sigils as files and folders. In @SigilSpecContent it should be described plainly as storage structure and serialization format only—the boring implementation detail of how a sigil is stored on disk, including its layout for language, affordances, invariants, and children—with any ontology or explanatory authority stripped away.
