# Decision Frame

Viewing: Application > SigilFolder > LanguageFile

## User

How would you tighten the language?

## Partner

I’d make it say only that a SigilFolder is the on-disk representation of a sigil, containing its `language.md`, affordance files, invariant files, and child sigil folders. That keeps it concrete and removes any suggestion that the folder itself is the deeper locus of identity or memory.

A tighter version would be: “A `SigilFolder` is the filesystem representation of a @sigil. It stores the sigil’s `language.md`, affordances, invariants, and child sigils as files and folders. It exists for persistence and editing, not as the source of the sigil’s experiential memory or conceptual identity.”
