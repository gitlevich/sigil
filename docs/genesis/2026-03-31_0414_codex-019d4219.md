I would measure it as a composite score, not one raw embedding number.

`C = 0.25 * boundary + 0.20 * affordance_coverage + 0.20 * refinement + 0.15 * sibling_separation + 0.10 * neighbor_contract + 0.10 * determinacy`

- `boundary`: embed each sigil from its definition, language, affordances, and contrasts. Score high when it is distinct from siblings, consistent with its own preferred contrast ranges, and not semantically leaking across the boundary.
- `affordance_coverage`: every parent affordance should be supported by at least one descendant path. Score by matching each affordance against child or leaf sigils that materially realize it; penalize uncovered affordances.
- `refinement`: a child should refine the parent, not drift away from it. Score parent-child embedding alignment along the path, with penalties when a child introduces meaning that is not grounded upstream.
- `sibling_separation`: siblings should occupy different semantic regions unless intentionally coupled. Score inter-sibling distance and penalize overlap.
- `neighbor_contract`: cross-sigil interaction should happen through explicit affordances and declared neighbor relationships only. Penalize undeclared dependencies, direct concept leakage, or missing exposed affordances.
- `determinacy`: for a leaf, prompt the AI multiple times and measure variance in structured outputs or implementation plans. Low variance means the sigil is specific enough to project deterministically.

### Assistant (04:58 UTC)

For the boundary metric to mean anything, a sigil has to be a real semantic region, not just a folder with a name.

**Boundary Metric**

A useful boundary metric is asking:

`Does this sigil define a distinct interior that is internally consistent, externally separated, and sufficient to realize its affordances?`

That is only meaningful if all of these are true:

- The observer is fixed.
  The affordances have to be “for whom?” stable. If the observer shifts, the boundary shifts.
- The level of abstraction is fixed.
  Siblings must be comparable peers, not one concrete mechanism next to one vague umbrella.
- The sigil has a stable responsibility.
  It should exist because it must expose or realize a coherent set of affordances, not because it is a convenient bucket.
- The interface is explicit.
  What is visible outside the sigil must be its affordances and declared neighbor relations, not hidden internal concepts.
- The contrast space is explicit.
  The boundary must be defined by named dimensions with preferred ranges, not by vague prose alone.
- The child decomposition is jointly sufficient.
  Children should together explain how the parent’s affordances become realizable.
- Siblings are meaningfully distinct.
  If two siblings occupy the same semantic region, the boundary is fake.

If those are not true, an embedding score is just measuring writing similarity, not boundary quality.

A good boundary score would usually combine four things:

- Internal tightness: the language, affordances, and contrasts of the sigil point to the same concept.
- External separation: the sigil is distinct from siblings except where explicit neighbor relations connect them.
- Realization support: the child sigils actually support the parent affordances.
- Leakage penalty: concepts that belong in siblings or neighbors are not doing quiet work inside this sigil.

So conceptually:

`boundary = tightness + support - sibling_overlap - leakage`

**How To Define Them**

A `sigil` should be defined as:
- a coherent interior a specific observer can inhabit
- a set of affordances it exposes in that context
- a set of contrasts whose preferred ranges define acceptable realizations
- a decomposition into up to five child sigils only when more internal affordances are needed

A `contrast` should be:
- a named dimension of variation that actually matters inside this sigil
- expressed with poles or a range, not just an adjective
- identity-defining: outside the preferred range, the sigil stops being “right”

Good contrasts:
- latency: immediate to deliberate
- visibility: ambient to focal
- autonomy: guided to self-directed

Bad contrasts:
- quality
- nice UX
- robustness
These are too broad unless sharpened.

An `affordance` should be:
- externally usable by the observer or by neighbors in the shared context
- action-shaped, not implementation-shaped
- something whose absence would mean this sigil is not doing its job

Good affordances:
- browse sigils
- open current context
- rename boundary
- compare neighbors

Bad affordances:
- store state
- call API
- render component
These are usually internal machinery, not observer-facing handles.

**Practical Rule**

A sigil is well-defined when you can say:

- what can be done with it
- what dimensions matter inside it
- what ranges are acceptable on those dimensions
- what children are needed to make those affordances real
- what must stay outside and only be reached through neighbors
