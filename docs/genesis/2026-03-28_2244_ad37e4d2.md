## Tauri

Tauri is a framework for building small, fast, secure desktop (and mobile) applications using web technologies for the frontend and Rust for the backend.

**Core architecture:**
- **Frontend**: Any web framework (React, Svelte, Vue, etc.) renders in a system webview (WebKit on macOS/Linux, WebView2 on Windows)
- **Backend**: Rust process handles system-level operations, file I/O, IPC, and native APIs
- **IPC bridge**: Frontend communicates with the Rust backend via a command/event system

**Key differences from Electron:**
- No bundled Chromium — uses the OS webview, so binaries are ~5-10 MB vs Electron's ~150+ MB
- Rust backend instead of Node.js — better performance and memory usage
- Stronger security model — explicit permission system, no full Node access from the renderer

**Tauri 2 (current):**
- Added mobile support (iOS/Android) alongside desktop
- Plugin-based architecture — capabilities like file system access, shell, dialogs are plugins
- Fine-grained permission system via `capabilities` in `tauri.conf.json` — you declare exactly what each window can access
- Commands are Rust functions annotated with `#[tauri::command]`, callable from JS via `invoke("command_name", { args })`

**Typical project structure:**
```
src-tauri/       # Rust backend
  src/
    main.rs      # Entry point
    lib.rs       # Commands, setup
  tauri.conf.json
  Cargo.toml
src/             # Frontend (React, etc.)
  App.tsx
  main.tsx
```

**Trade-offs to know:**
- Webview rendering can differ slightly across OSes (unlike Electron's consistent Chromium)
- Rust has a steeper learning curve than Node.js
- Ecosystem is younger than Electron's, though growing fast

It's a strong choice when you want native-feeling apps with small bundles and don't need pixel-perfect cross-platform rendering consistency.

Tauri's core domain concepts:

- **Window** — a native OS window hosting a webview
- **Webview** — the embedded browser rendering your frontend
- **Command** — a Rust function exposed to the frontend via `#[tauri::command]`, invoked with `invoke()`
- **Event** — async message passing between frontend and backend (emit/listen pattern)
- **State** — managed Rust state shared across commands via `tauri::State<T>`
- **Plugin** — modular extension (e.g. `tauri-plugin-fs`, `tauri-plugin-shell`) that bundles commands, events, and permissions
- **Capability** — a named permission set declaring what a window is allowed to access (file system, shell, etc.)
- **Permission** — a granular allow/deny rule within a capability
- **App Handle** (`AppHandle`) — reference to the running app instance, used to access windows, state, and emit events from anywhere in the backend
- **Manager** — trait implemented by `AppHandle` and `Window`, providing access to app resources
- **IPC** — the inter-process communication bridge between the webview (JS) and the Rust core
- **Scope** — restricts what paths/URLs a permission grants access to (e.g. only `$APPDATA/**`)
- **Resource** — app-bundled assets accessible at runtime
- **Sidecar** — an external binary bundled and launched alongside your app
