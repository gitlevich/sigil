As of March 30, 2026, the cleanest supported path on macOS is: install the CLI, run the onboarding wizard, keep the Gateway local with token auth, then use the Control UI for the first chat. If you already use Codex, OpenClaw can reuse that login during setup.

1. Install OpenClaw.

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

The installer handles Node 22+ if needed. If it does not drop you into setup automatically, run:

```bash
openclaw onboard --install-daemon
```

2. In the onboarding wizard, pick `Local` / `This Mac`, keep the default workspace at `~/.openclaw/workspace`, keep Gateway auth in `token` mode, and say yes to the daemon install. For model auth, the simplest choices are:
`OpenAI Code subscription` if you want it to reuse your existing Codex login.
`OpenAI API key` if you want direct API billing.
`Anthropic API key` if you want Claude directly.

3. Verify the install.

```bash
openclaw gateway status
openclaw health
openclaw dashboard
```

The local Control UI is normally [http://127.0.0.1:18789/](http://127.0.0.1:18789/).

4. Configure the agent in `~/.openclaw/openclaw.json`. Most config changes hot-apply automatically; `gateway.*` and `plugins` changes are the main ones that trigger restart behavior.

If you use Codex subscription auth, a minimal config is:

```json5
{
  agents: {
    defaults: {
      workspace: "~/.openclaw/workspace",
      model: { primary: "openai-codex/gpt-5.4" }
    }
  }
}
```

If you use an OpenAI API key instead, the simple version is:

```json5
{
  env: { OPENAI_API_KEY: "sk-..." },
  agents: {
    defaults: {
      workspace: "~/.openclaw/workspace",
      model: { primary: "openai/gpt-5.4" }
    }
  }
}
```

If you do not want plaintext keys in config, export the key first and onboard with env-backed secret refs:

```bash
export OPENAI_API_KEY="sk-..."
openclaw onboard --install-daemon --auth-choice openai-api-key --secret-input-mode ref
```

5. If you want chat outside the browser, Telegram is the fastest channel to add. Put this in `~/.openclaw/openclaw.json`:

```json5
{
  channels: {
    telegram: {
      enabled: true,
      botToken: "123:abc",
      dmPolicy: "pairing",
      groups: { "*": { requireMention: true } }
    }
  }
}
```

Then start and approve the first DM:

```bash
openclaw gateway
openclaw pairing list telegram
openclaw pairing approve telegram <CODE>
```

Two practical defaults matter:
Keep the dashboard on localhost, Tailscale, or an SSH tunnel. It is an admin surface and should not be exposed publicly.
Keep token auth enabled even on loopback.
